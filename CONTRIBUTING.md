# Contributing to minio-rust

Thanks for helping improve `minio-rust`.

## Development workflow

1. Fork the repository and create a focused feature branch.
2. Make the smallest change that fully solves the problem.
3. Add or update tests when behavior changes.
4. Run the local checks before opening a pull request:

```sh
cargo fmt
cargo test
```

## Pull request expectations

- Keep pull requests small and reviewable.
- Explain what changed and why.
- Call out tradeoffs, follow-up work, and any known limitations.
- Avoid mixing refactors with behavior changes unless they are tightly related.
